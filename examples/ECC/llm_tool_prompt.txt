================================================================
6. Function Notes
================================================================

Below is a short usage guide for the main functions. Use these descriptions to
choose valid arguments and legal construction sequences.

Constructors:

`from_generator_matrix(matrix)`
- Build a code directly from a generator matrix.
- `matrix` should be a 2D binary array-like object.
- Good when you already know a generator matrix explicitly.

`from_parity_check_matrix_binary(H)`
- Build a code from a binary parity-check matrix.
- Good when the code is easier to describe by parity equations.

`build_family(family_name, **kwargs)`
- Build a standard deterministic family.
- Use one of:
  - `"cyclic_binary"`
  - `"constacyclic_binary"`
  - `"hamming_binary"`
  - `"simplex_binary"`
  - `"double_circulant_binary"`

Family-specific notes:

`cyclic_code_binary(n, generator_poly)`
- `n` is the code length.
- `generator_poly` uses low-to-high coefficient order.
- Use when you want a cyclic code defined by a generator polynomial.

`constacyclic_code_binary(n, generator_poly, shift_constant)`
- Same style as cyclic code construction.
- In this implementation, `shift_constant` must be `1`.

`hamming_code_binary(r)`
- `r` is the redundancy parameter.
- Produces a binary Hamming code of length `2^r - 1`.

`simplex_code_binary(r)`
- Produces the binary simplex code, dual to the Hamming code with the same `r`.

`double_circulant_code_binary(first_row)`
- `first_row` is a binary vector.
- Produces a code with generator matrix `[I | C]`, where `C` is circulant.

Basic transformations:

`augment_code(code)`
- Add the all-ones vector if it is not already in the code.

`code_complement(code, subcode)`
- Return a complement `C2` such that `code = subcode + C2`.
- Constraint:
  - `subcode` must be a subcode of `code`.

`direct_sum(c1, c2)`
- Form a direct sum by block-concatenating coordinates.

`direct_sum_many(codes)`
- Repeated direct sum over a list of codes.

`direct_product(c1, c2)` or `product_code(c1, c2)`
- Use the Kronecker-product construction on generator matrices.

`extend_code(code)`
- Add one parity coordinate.

`extend_code_n(code, n)`
- Apply `extend_code` repeatedly.
- `n` must be non-negative.

`pad_code(code, n)`
- Append `n` zero coordinates to the right.

`lengthen_code(code)`
- Equivalent to `extend_code(augment_code(code))`.

Coordinate and Plotkin operations:

`plotkin_sum(c1, c2)`
- Plotkin-style construction from two codes.
- Lengths are padded automatically if needed.

`plotkin_sum_three(c1, c2, c3, a=-1)`
- Three-way Plotkin-style construction.
- In GF(2), `a` must be odd.

`puncture_code(code, i)`
- Delete coordinate `i`.
- Coordinates are 0-based.

`puncture_code_set(code, positions=[...])`
- Delete several coordinates.
- Positions must be distinct and valid.

`shorten_code(code, i, max_k_enum=...)`
- Keep only codewords that are zero at coordinate `i`, then puncture that coordinate.

`shorten_code_set(code, positions=[...], max_k_enum=...)`
- Shorten on multiple coordinates.
- This enumerates codewords, so keep `max_k_enum` reasonably small.

Expurgation, duality, intersection:

`expurgate_code_odd_weight(code, max_k_enum=...)`
- Keep the even-weight subcode.
- This enumerates codewords.

`expurgate_code_list(code, words_to_delete=[...], max_k_enum=...)`
- Remove the subcode generated by the listed codewords.
- The listed codewords must belong to the input code.

`expurgate_weight_code(code, weight=..., max_k_enum=...)`
- Find a non-zero codeword of the given weight and remove the 1-dimensional subcode it generates.

`dual_code(code)`
- Return the dual code `C^perp`.

`intersection_code(c1, c2)`
- Return the intersection `c1 ∩ c2`.

Advanced constructions:

`concatenate_codes(c1, c2)`
- Pairwise row concatenation of generator rows.

`juxtaposition(c1, c2)`
- Horizontal join `[A | B]`.
- Constraint:
  - `c1.k == c2.k`

`concatenated_code(outer, inner)`
- Concatenated code construction.
- Constraint:
  - this implementation currently supports only `inner.k == 1`

`construction_x(c1, c2, c3)`
- Construction X style lift.
- Constraint:
  - `c2 <= c1`
  - `c3.k` must be large enough for the quotient part

`construction_x_chain(codes, c)`
- Apply Construction X to a descending chain.
- Constraint:
  - `codes[i+1] <= codes[i]`

`construction_x3(c1, c2, c3, d1, d2)`
- Construction X3 style lift.
- Constraint:
  - `c3 <= c2 <= c1`

`construction_x3u(c1, c2, c3, d1, d2)`
- Return a pair of lifted codes in an X3u-style pattern.

`construction_xx(c1, c2, c3, d2, d3)`
- Construction XX style lift.
- Constraint:
  - `c2 <= c1`
  - `c3 <= c1`

`zinoviev_code(inner, outer)`
- Simplified generalized concatenation.
- Constraints:
  - `inner[0] <= inner[1] <= ...`
  - inner dimensions must strictly increase
  - all inner codes have the same length
  - all outer codes have the same length

`construction_y1(code, weight=None, max_k_enum=...)`
- Use a support of a dual codeword, then shorten.
- If `weight` is omitted, use a minimum non-zero dual weight.
- This enumerates codewords.

Enumeration-sensitive functions:

- `shorten_code`
- `shorten_code_set`
- `expurgate_code_odd_weight`
- `expurgate_code_list`
- `expurgate_weight_code`
- `construction_y1`

For these functions, keep `max_k_enum` reasonably small.
